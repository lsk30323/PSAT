# 자료해석 — 조건 적용·보고서형 [학습지]

> 국회직 8급 PSAT 2차 자료해석(20문항·45분) 대비. 「개념 이해 → 실전 문제 → 정답 → 해설」 순서로 학습한다. 난도 기준은 **민경채~국가직 7급 밴드**. 아래 문제의 숫자·소재는 학습용으로 창작한 것이며 실제 기출 수치가 아니다.

---

## 개념 이해 (핵심 정리)

### 유형 정의
표·그림 자료에 **조건(각주 산식·대소·증감·합계)이나 보고서(서술문)**가 결합된 묶음 유형. 4개 하위 갈래로 나뉜다.

| 하위 유형 | 묻는 방식 |
|---|---|
| **보고서 부합형** | 보고서 속 통계 서술의 정오를 ㄱ/ㄴ/ㄷ로 판단 |
| **추가 필요자료형** | 보고서 작성에 "추가로 필요한 자료"를 보기에서 모두 고르기 |
| **조건 매칭(짝짓기)형** | 각주·조건을 적용해 익명 A~E를 실제 명칭에 짝짓기 |
| **각주 산식 적용·빈칸형** | 정의식(1인당·점유율·증가율)을 빈칸·미지값에 대입·역산 |

자료해석 20문항 중 **약 4~6문항(1/4~1/3)**을 차지하며, 시험 앞부분(1·2번)부터 깔려 **초반 페이스를 좌우**한다.

### 핵심 스킬 — "조건을 수식·체크리스트로 정식화 → 경계값만 검증"
보고서형의 본질은 계산력이 아니라 **조건 번역 + 검증 순서 설계**다.

1. **선(先) 정식화** — 조건을 읽는 즉시 기호로.
   - "A는 B보다 크다" → `A > B`  /  "매년 증가" → `xₜ < xₜ₊₁ (모든 t)`  /  "전체의 절반 이상" → `값 ≥ 0.5 × 합계`.
   - 각주 정의식(1인당 = 총량/인구, 점유율 = 해당값/전체×100)은 **문제 옆에 그대로 베껴 적는다.**
2. **AND/OR·전칭 표시** — 접속사에 동그라미(그리고=∧, 또는=∨). "매년/모든/항상"(전칭)은 **반례 1개**면 깨진다.
3. **충족 쉬운 조건부터 후보 축소** — 매칭형은 계산 없이 걸러지는 조건(최대·최소·유일하게 감소)으로 선지를 먼저 배제. **모든 항목을 계산하지 않는다.**
4. **경계값만 확인** — "절반 이상" "30% 초과"처럼 임계선이 있으면 `임계 × 합계`로 **기준선 1개**를 만들어 대소만 비교. 명백히 크거나 작은 건 어림산 통과.
5. **선지 역이용** — 짝짓기·빈칸은 5지선다 보기가 조합이다. 한 조건으로 후보가 2개로 줄면 나머지는 보기에서 역추적해 계산을 0~1회로 끝낸다.

### 계산 스킬
- **나눗셈은 곱셈으로**: `a/b ≥ 0.3?` → `a ≥ 0.3b?` (분수 비교 회피).
- **분수 대소는 교차곱**: `a/b vs c/d` → `a·d vs c·b`.
- **빈칸은 합계로 역산**: `빈칸 = 합계 − 나머지의 합`.
- **%p와 % 분리**: "%p 증가"는 **차**(15%−12%=3%p), "% 증가"는 **비**(3/12=25%).
- **단위 통일**: 천/만/억, 표 단위와 보고서 단위를 먼저 맞춘 뒤 비교.

### 대표 함정 (Top 5) — ✗ 틀리는 처리 / ✓ 막는 동작
1. **AND/OR 혼동** — ✗ "그리고"를 "또는"으로 읽어 한 조건만 맞는 항목까지 통과. ✓ ∧이면 **두 조건 모두 통과한 교집합만** 남긴다.
2. **전칭("매년/모든") 무시** — ✗ 우상향 추세만 보고 ○. ✓ **반례 사냥 모드** — 인접 두 해 부호만 훑어 감소·동일 구간 1개라도 있으면 즉시 ✗.
3. **반올림·소수 처리** — ✗ 보고서 "약 12.3%"와 표 계산 "12.27%"가 다르다고 ✗ 판정. ✓ **"약·내외·가량"은 반올림 허용 신호**(둘째 자리 차이 일치 처리). "정확히·이상·초과"일 때만 원값으로 엄격 검증.
4. **단위 환산 누락** — ✗ 표는 천 명, 보고서는 만 명인데 숫자만 비교(10배 어긋남). ✓ 보고서 수치를 **표 단위로 먼저 환산**.
5. **추가자료형 "이미 있는 자료" 재선택** — ✗ "있으면 좋은" 자료를 정답으로. ✓ **2단 필터**: (1) 보고서가 실제로 요구하는가? (2) 그 값이 **현재 표에 없는가?** **둘 다 ✓일 때만** 정답.

### 핵심 한 줄
> **'초과(>)'와 '이상(≥)'은 한 글자가 정답을 가른다.** 임계선과 정확히 같은 값이 있으면 그 항목의 포함 여부가 답을 결정하므로, 조건 문장의 어미를 반드시 부등호로 옮겨 적어라.

### 문제 1. (각주 산식 적용·빈칸 역산형)

다음 〈표〉는 만보기 앱 4개사와 기타 앱의 월간 활성 사용자 수(MAU)와 시장 점유율을 정리한 것이다.

**〈표〉 만보기 앱 시장 현황 (전체 MAU = 4,000천 명)**

| 앱 | MAU(천 명) | 점유율(%) |
|---|---:|---:|
| 워크메이트 | 1,100 | ( A ) |
| 핏스텝 | ( B ) | 22.0 |
| 데일리워크 | 720 | 18.0 |
| 헬스플러스 | 500 | ( C ) |
| 기타 | ( D ) | 20.0 |
| **합계** | **4,000** | **100.0** |

※ 점유율(%) = (해당 앱 MAU / 전체 MAU) × 100. 모든 값은 나누어떨어진다.

빈칸 A~D에 들어갈 값으로 **옳지 않은** 것은?

① A = 27.5
② B = 880
③ C = 12.5
④ D = 720
⑤ 핏스텝의 점유율(22.0%)을 MAU로 환산하면 880천 명이다

### 문제 2. (조건 매칭·짝짓기형)

다음 〈표〉는 워크온이 운영하는 4개 지역(서울·부산·대구·광주)의 헬스 앱 가입자 현황을 익명 A~D로 나타낸 것이다. 〈조건〉을 모두 만족하도록 A~D를 짝지을 때 옳은 것은?

**〈표〉 지역별 가입자 현황**

| ② | A | B | D | C |

### 문제 3. (비중·구성비 임계 조건형, ㄱ/ㄴ/ㄷ)

다음 〈표〉는 워크온 적립 포인트의 사용처별 사용 건수를 나타낸 것이다. 사용처는 아래 4개로 한정된다.

**〈표〉 포인트 사용처별 사용 건수 (합계 = 6,000건)**

| 사용처 | 사용 건수 |
|---|---:|
| 기프티콘 | ( ㉮ ) |
| 헬스용품 | 1,500 |
| 기부 | 900 |
| 통신비 | 1,800 |
| **합계** | **6,000** |

※ 구성비(%) = (해당 사용처 건수 / 합계) × 100.

빈칸 ㉮를 채운 뒤, 다음 〈보기〉 중 옳은 것만을 모두 고른 것은?

**〈보기〉**
- ㄱ. 구성비가 30%를 **초과**하는 사용처가 있다.
- ㄴ. 구성비가 30% **이상**인 사용처는 2개이다.
- ㄷ. 기프티콘의 구성비는 통신비보다 높다.

① ㄱ  ② ㄴ  ③ ㄷ  ④ ㄱ, ㄴ  ⑤ ㄴ, ㄷ

## 정답

`1. ④   2. ②   3. ②`

## 해설

### 문제 1 — 정답 ④

**조건 → 수식 번역**: 각주를 그대로 베낀다 → `점유율 = MAU / 4,000 × 100`. 빈칸은 ① 점유율 칸은 `MAU/40`, ② MAU 칸은 `점유율 × 40`으로 양방향 역산하거나, ③ 합계(MAU 4,000 / 점유율 100%)로 한 칸을 역산한다.

**빈칸 역산 (실제 계산)**
- **A (워크메이트 점유율)** = `1,100 / 4,000 × 100 = 27.5%` → ① 옳음.
- **B (핏스텝 MAU)** = `0.22 × 4,000 = 880`천 명 → ② 옳음. (⑤도 같은 값을 묻는 것이므로 옳음.)
- **C (헬스플러스 점유율)** = `500 / 4,000 × 100 = 12.5%` → ③ 옳음.
- **D (기타 MAU)** — **합계로 역산**: `4,000 − (1,100 + 880 + 720 + 500) = 4,000 − 3,200 = 800`천 명.
  - 검산(점유율로도): `0.20 × 4,000 = 800`. 두 경로 모두 **800**.
  - 선지는 **D = 720**이라 했으므로 → **④ 옳지 않음 (정답).**

**함정 회피**: D를 점유율(20%)이 아니라 데일리워크 MAU(720)와 헷갈려 그대로 옮기면 오답에 걸린다. **빈칸은 반드시 "합계 − 나머지" 또는 각주식으로 독립 산출**하고, 다른 행의 숫자를 베끼지 않는다. 검산을 두 경로(합계·각주식)로 돌리면 720이 끼어들 자리가 없음을 즉시 안다.

### 문제 2 — 정답 ②

**조건 → 강한 순서로 적용** (계산 없이 후보가 즉시 확정되는 조건 먼저).

1. **㉠ 서울 = 가입자 최다** (강함, 즉시 확정): 가입자 520 > 450 > 310 > 280 → 최댓값은 **A = 서울**. 선지에서 서울 ≠ A인 ③(서울 C)·⑤(서울 D) **소거**. → 남은 후보 ①·②·④.
2. **㉡ 부산 = 유일하게 감소** (증감률 < 0 인 곳): 증감률 부호만 보면 A(+8)·C(+12)·D(+5)는 증가, **B(−3)만 감소** → 유일 → **B = 부산**. 선지에서 부산 ≠ B인 ④(부산 D) **소거**. → 남은 후보 ①·②.
3. **㉢ 대구 걸음 수 > 광주** (남은 C·D 배정): A·B가 서울·부산으로 빠졌으므로 대구·광주는 **{C, D}**. 걸음 수 비교 → **D = 7,900 > C = 6,800** → **대구 = D, 광주 = C**.
   - 선지 ① (대구 C, 광주 D)은 ㉢에 위배 → 소거. 선지 ② (대구 D, 광주 C)이 부합.

**결론**: 서울 A · 부산 B · 대구 D · 광주 C → **② (정답).**

**함정 회피**: 약한 조건(㉢ 걸음 수 비교)부터 풀면 비교 대상이 4개로 늘어 계산이 폭증한다. **㉠(최댓값)·㉡(유일 감소)처럼 후보를 1개로 확정하는 강한 조건을 선행**하면, 마지막에 남는 둘만 비교(계산 1회)로 끝난다. 증감률은 값이 아니라 **부호(+/−)만** 보면 된다.

### 문제 3 — 정답 ②

**빈칸 역산**: `㉮ = 합계 − 나머지 = 6,000 − (1,500 + 900 + 1,800) = 6,000 − 4,200 = 1,800`건.

**구성비 (실제 계산)** — 합계 6,000으로 나눈다.
| 사용처 | 건수 | 구성비 |
|---|---:|---:|
| 기프티콘 | 1,800 | `1,800/6,000 = 30.0%` |
| 헬스용품 | 1,500 | `1,500/6,000 = 25.0%` |
| 기부 | 900 | `900/6,000 = 15.0%` |
| 통신비 | 1,800 | `1,800/6,000 = 30.0%` |

**임계선 1개로 판정**: 30% 경계 = `0.30 × 6,000 = 1,800`건. (각 항목을 %로 나누지 않고 1,800과 대소만 봐도 된다.)

- **ㄱ. 30% 초과(`> 1,800`) 사용처가 있다** — 기프티콘·통신비 모두 정확히 **1,800 = 1,800**이라 **초과가 아님(등호 불포함)**. 1,800을 넘는 항목 0개 → **거짓**.
- **ㄴ. 30% 이상(`≥ 1,800`)인 사용처는 2개** — 기프티콘 1,800 ≥ 1,800 (○), 통신비 1,800 ≥ 1,800 (○), 헬스용품 1,500·기부 900은 미달 → **정확히 2개** → **참**.
- **ㄷ. 기프티콘 구성비 > 통신비** — 둘 다 30.0%(건수 1,800으로 동일) → 높지 않고 **같음** → **거짓**.

**결론**: 참인 보기는 **ㄴ 하나** → **② (정답).**

**함정 회피**: 핵심은 **'초과 vs 이상'의 경계 함정**이다. 기프티콘·통신비가 임계선(1,800)과 **정확히 같으므로**, 어미가 "초과(>)"인 ㄱ에서는 **둘 다 탈락**(거짓), "이상(≥)"인 ㄴ에서는 **둘 다 포함**(참)된다. 만약 ㄱ이 "30% 이상"이었다면 ㄱ도 참이 되어 답이 바뀐다 — **조건 문장의 어미를 부등호로 정확히 옮겨 적는 것**이 단일 정답을 가른다. 또한 ㄷ처럼 동률(30%=30%)은 "높다"가 아니라 "같다"이므로 거짓 처리한다.
