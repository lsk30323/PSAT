## 1. 개념 이해 (핵심 정리)

### 유형 정의
표(원자료)를 그래프로 옮긴 5개 선지 중 **바르게 나타낸 것** 또는 **옳지 않게 나타낸 것** 하나를 고르는 유형. 막대·꺾은선·원(파이)·누적막대·100% 누적·이중축 등 다양한 그래프가 등장한다. 복잡한 나눗셈이 거의 없어 **"눈대중 문제(눈으로 판단)"**로 불리며, 자료해석에서 **시간을 벌어주는 가성비 득점원**이다.

### 핵심 스킬 — 전수 검산 금지, '튀는 지점'만 검증
5개 선지의 모든 점을 표와 대조하면 시간이 폭발한다. **절대 전부 계산하지 않는다.** 다음 순서로 솎아낸다(소거).

1. **(0~10초) 스캔** — 그래프 종류·세로축 단위(천 명? %? 억 원?)·시작점(0인가?)·가로축 항목 순서를 먼저 본다. 여기서 1~2개가 바로 탈락하기도 한다.
2. **(추세 소거)** 증감 방향이 어긋나는 선지 제거. 표가 "증가→감소"인데 그래프가 계속 증가하면 즉시 OUT. 점이 아니라 **꺾이는 방향**만 본다.
3. **(극값 소거)** 최댓값·최솟값 위치가 틀린 선지 제거. **1등·꼴찌 막대만** 확인.
4. **(교차·역전 소거)** 두 계열이 **교차하는 지점**·대소 역전 지점이 표와 맞는지만 확인.
5. **(남은 1~2개 정밀)** 그제서야 의심 점 한두 개만 어림셈. 정확한 나눗셈 대신 **반올림·자릿수·"절반보다 큰가/작은가"** 식으로.

> 목표 시간 **문항당 1분 30초~2분**. 변환형은 평균보다 빨리 끝내고 계산형에 시간을 몰아준다.

### 전략 한 줄 요약
- **긍정형("바르게 나타낸 것")**: 5개 중 4개에 흠이 있다 → 흠(방향·극값·수치)을 찾아 **4개를 소거**하면 남는 1개가 정답.
- **부정형("옳지 않게 나타낸 것")**: 4개는 옳고 **틀린 1개**가 정답 → 가장 의심스러운 곳(축조작·순서이상·유일 감소구간)부터 정밀 대조해 **불일치 1점**을 찾으면 즉시 확정(나머지 검증 불필요).

### 대표 함정 (출제자가 노리는 지점)
1. **누적그래프 ↔ 일반(병렬)그래프 혼동** — 누적막대 총높이는 "구간 값"이 아니라 "합계". 개별 값을 누적 높이로 착각하면 전부 틀린다.
2. **세로축 시작점 조작(0이 아님)·물결선** — 작은 차이를 크게 보이게. **축 눈금부터 확인.**
3. **축 단위·배율 바꿔치기** — 표는 "천 명", 그래프는 "만 명"으로 10배/0.1배 왜곡.
4. **항목 순서 바꿔치기** — 가로축 항목(연도·지역) 순서를 표와 다르게, 또는 인접 두 값을 swap.
5. **비중(%) ↔ 절대값 혼동** — 절대 수가 늘어도 점유율은 줄 수 있음. 모집단 곱(비율×N) 누락.
6. **증가율 ↔ 증가량 혼동** — 증가량(차이)을 증가율 자리에 그대로. 부호는 맞아도 크기 순서가 다를 수 있음.
7. **두 계열 범례(남/여 등) 라벨 뒤바꿈** — 교차 위치는 맞는데 선 색·라벨만 swap.

## 2. 실전 문제

> 표↔그래프 매칭은 지면에 그래프를 그릴 수 없으므로, 표는 텍스트로 제시하고 **각 보기 ①~⑤는 그래프를 말로 서술**한다(예: "막대그래프: 2021=120, 2022=150"). 수치·소재는 학습용 창작.

**〈표〉 연도별 일일 평균 걸음 수** (단위: 보)

| 연도 | 2019 | 2020 | 2021 | 2022 | 2023 |
|------|------|------|------|------|------|
| 걸음 수 | 8,000 | 6,500 | 9,200 | 8,800 | 10,500 |

① 막대그래프(2019~2023): 8,000 / 7,500 / 9,200 / 8,800 / 10,500
② 막대그래프(2019~2023): 8,000 / 6,500 / 9,200 / 10,500 / 8,800
③ 막대그래프(2019~2023): 9,200 / 6,500 / 8,000 / 8,800 / 10,500
④ 막대그래프(2019~2023): 8,000 / 6,500 / 9,200 / 8,800 / 10,500
⑤ 막대그래프(2019~2023): 8,000 / 6,500 / 8,800 / 9,200 / 10,500

**〈표〉 운동 종류별 사용자 수** (단위: 명)

### 문제 3.

다음 〈표〉는 어느 헬스 앱의 **월별 신규 가입자 수**를 성별로 나타낸 것이다. 〈표〉를 **남·여 두 개의 꺾은선그래프로 바르게 나타낸 것**은?

**〈표〉 월별 신규 가입자 수** (단위: 명)

| 월 | 1월 | 2월 | 3월 | 4월 |
|----|-----|-----|-----|-----|
| 남 | 400 | 500 | 700 | 600 |
| 여 | 600 | 550 | 500 | 750 |

① 남 선: 400 / 500 / 700 / 600,  여 선: 600 / 550 / 500 / 750 — 두 선은 **2월과 3월 사이**에서 1회 교차(여 우위 → 남 우위)하고, 4월에는 **여 선이 더 높다.**
② 남 선: 400 / 500 / 700 / 600,  여 선: 600 / 550 / 500 / 750 — 두 선은 **3월과 4월 사이**에서 1회 교차하고, 4월에는 남 선이 더 높다.
③ 남 선: 600 / 550 / 500 / 750,  여 선: 400 / 500 / 700 / 600 — 두 선은 2월과 3월 사이에서 1회 교차하고, 4월에는 여 선이 더 높다.
④ 남 선: 400 / 500 / 700 / 600,  여 선: 600 / 550 / 500 / 750 — 남 선의 최고점은 **4월(600)**, 여 선의 최고점은 4월(750)이다.
⑤ 남 선: 400 / 500 / 700 / 600,  여 선: 600 / 550 / 500 / 750 — 두 선은 **2월과 3월 사이**에서 1회 교차(여 우위 → 남 우위)하고, 4월에는 **여 선(750)이 남 선(600)보다 높다.**

## 4. 해설

### 문제 1. 정답 ④

**튀는 지점: 증감 방향 시퀀스 + 극값 위치.** 표를 점이 아니라 **방향**으로 먼저 읽는다.

- 표 방향 시퀀스: 8,000 → 6,500(**↓**) → 9,200(**↑**) → 8,800(**↓**) → 10,500(**↑**). 즉 패턴은 `↓ ↑ ↓ ↑`.
- 극값: 최댓값 = **10,500(2023)**, 최솟값 = **6,500(2020)**.

| 선지 | 어긋난 '튀는 지점' | 판정 |
|------|--------------------|------|
| ① | 2020을 7,500으로 표기(표 6,500) → 1·2번째 값 사이 변화가 ↓이긴 하나 **최솟값 위치/수치 불일치**(6,500≠7,500) | 소거 |
| ② | 2022=10,500, 2023=8,800으로 뒤바꿈 → **최댓값이 2022에 찍힘**(표 최댓값은 2023). 끝 두 값 swap | 소거 |
| ③ | 2019=9,200, 2021=8,000으로 swap → **최댓값(9,200)이 2019에**, 첫 구간이 ↑로 바뀜(표는 첫 구간 ↓) | 소거 |
| ④ | 8,000 / 6,500 / 9,200 / 8,800 / 10,500 → 방향 `↓↑↓↑` 일치, 최대 2023·최소 2020 일치, 다섯 값 모두 일치 | **정답** |
| ⑤ | 2021=8,800, 2022=9,200으로 swap → 표는 2021→2022가 **감소(↓)**인데 ⑤는 **증가(↑)**. 방향 1개 불일치 | 소거 |

**함정 회피:** ⑤는 가운데 두 값만 살짝 바꾼 전형적 함정으로, 전체를 보면 멀쩡해 보인다. 그러나 "2021→2022 방향"이라는 **튀는 한 점의 방향**만 보면 8,800→9,200(↑)으로 표(↓)와 어긋나 즉시 소거된다. 8,200/7,400 식 나눗셈은 단 한 번도 필요 없다.

### 문제 2. 정답 ②

**튀는 지점: 각 조각의 구성비(비중) 어림.** 합계 2,000명을 기준으로 비율을 환산한다(전수 계산 아님, 한 조각씩).

- 걷기 900/2,000 = **45%**
- 러닝 600/2,000 = **30%**
- 자전거 300/2,000 = **15%**
- 수영 200/2,000 = **10%**
- (검산) 45+30+15+10 = **100%** ✔

부정형("옳지 않은 것")이므로 4개는 옳고 1개만 틀리다. 한 조각씩 대조한다.

| 선지 | 표 기준 정답 비중 | 선지 표기 | 판정 |
|------|------------------|-----------|------|
| ① 걷기 | 45% | 45.0% | 옳음 |
| ② 러닝 | **30%** | **35.0%** | **불일치 → 정답(틀린 것)** |
| ③ 자전거 | 15% | 15.0% | 옳음 |
| ④ 수영 | 10% | 10.0% | 옳음 |
| ⑤ 네 조각 | 45/30/15/10, 합 100% | 45.0/30.0/15.0/10.0, 합 100% | 옳음 |

**함정 회피:** ⑤는 네 조각을 한꺼번에 보여주며 합이 100%로 딱 맞아 "맞다"고 넘기기 쉽다. 출제자는 이렇게 멀쩡한 종합 선지로 시선을 끌고, **단일 항목 선지(②)에 한 곳만** 손댄다(러닝 30%→35%). 러닝은 600/2,000=30%가 정확하므로 35%는 명백한 오류다. 만약 ②가 옳다면 합이 105%가 되어 ⑤(합 100%)와 모순되므로, **두 선지를 교차 검증**해도 ②가 유일한 틀린 선지임이 확정된다.

### 문제 3. 정답 ⑤

**튀는 지점: 두 선의 교차 위치 + 4월 끝점 대소.** 점 값이 아니라 **남>여 / 여>남 대소 관계**만 추적한다.

- 1월: 남 400 < 여 600 → **여 우위**
- 2월: 남 500 < 여 550 → **여 우위**
- 3월: 남 700 > 여 500 → **남 우위**
- 4월: 남 600 < 여 750 → **여 우위**

대소 관계 변화: 여 → 여 → **남** → 여. 즉 **2월~3월 사이에서 한 번(여→남), 3월~4월 사이에서 한 번(남→여)** 으로 두 선은 **총 2회 교차**한다. 4월 끝점은 **여(750) > 남(600)**.

| 선지 | 핵심 단서 점검 | 판정 |
|------|----------------|------|
| ① | "2~3월 1회 교차"는 맞으나, "4월에 여 선이 더 높다"면서 **교차 횟수를 1회로만** 서술 → 표는 3~4월에 다시 역전(남→여)되어 **2회 교차**. 교차 구조 불일치 | 소거 |
| ② | 교차를 **3~4월 사이**로만 보고 "4월에 남이 더 높다"고 함 → 표 4월은 남 600 < 여 750(**여가 높음**). 끝점 대소 불일치 | 소거 |
| ③ | 남·여 데이터 계열이 **통째로 뒤바뀜**(남 선에 600/550/500/750) → **범례 라벨 swap** | 소거 |
| ④ | "남 선의 최고점은 4월(600)"이라 함 → 남의 최고점은 **3월(700)**. 극값 위치·수치 불일치 | 소거 |
| ⑤ | 계열 값 정확, 2~3월 사이 1회 교차(여→남) 명시, 4월에 여(750)>남(600) 명시 → 표와 일치 | **정답** |

**함정 회피:** ③은 교차·증감 모양이 표와 똑같아 보이지만 **남/여 라벨이 통째로 뒤바뀐** 범례 swap 함정이다(남 선이 1월 600에서 시작하는지 400에서 시작하는지만 보면 즉시 걸린다). ②·④는 "교차는 그럴듯한데 끝점 또는 극값이 1칸 어긋난" 전형이다. ⑤만 계열·교차·끝점이 모두 일치한다.

## 5. 한 줄 마무리
변환형은 **전수 검산 절대 금지**. ① 증감 방향, ② 극값(1등·꼴찌), ③ 교차/역전, ④ 축·단위, ⑤ 항목 순서 — 이 5대 '튀는 지점'으로 소거하면 5개 선지 중 손으로 계산하는 칸은 **1~2개**뿐이다. 누적·축조작·항목순서(범례 swap) 3대 함정만 외워도 정답률이 크게 오른다. 합격선 80점 돌파를 위해 이 유형은 **무실점 목표**.
