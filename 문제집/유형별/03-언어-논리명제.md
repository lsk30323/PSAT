# 언어논리 — 논리·명제(형식논리) 학습지

> 국회직 8급 PSAT 대비 · 「개념 이해 → 실전 문제 → 정답 → 해설」
> 난도: 민경채 ~ 국가직 7급 수준. 모든 문항은 논리적으로 **유일한 정답**이 도출되도록 설계·검증함.

---

## 개념 이해 (핵심 정리)

### 1. 유형 정의
주어진 **조건문·정언명제**만으로 "반드시 참/반드시 거짓/알 수 없는 것"을 가려내는 **연역추론** 유형. 지문 독해가 아니라 **기호 조작**이 본질이다. 세 하위 유형이 골고루 나온다.

1. **명제논리** — 조건문(`p→q`)의 대우·연쇄로 결론 도출
2. **참/거짓 진술 추리(진실게임)** — "정확히 n명만 거짓" 같은 메타조건으로 진위 확정
3. **정언·양화 추론** — "모든/어떤(전칭/특칭)" 명제의 포함관계·존재 추론

### 2. 핵심 스킬

**(1) 명제 기호화**
| 한국어 표현 | 기호 | 비고 |
|---|---|---|
| "A이면 B" | `A→B` | A = 충분조건(화살표 꼬리) |
| "B이려면 A가 필요" / "A여야만 B" / "A 없이는 B 불가" | `B→A` | A = **필요조건(화살표 머리)** |
| "A 그리고 B" | `A∧B` | |
| "A 또는 B" | `A∨B` | PSAT는 **포함적 OR**(둘 중 적어도 하나) |
| "모든 S는 P" (전칭 A) | `S⊆P` | 영역을 **비움**(빗금) |
| "어떤 S는 P" (특칭 I) | `∃x: S∧P` | 존재(×)를 **찍음** |

**(2) 대우 · 연쇄**
- 대우: `A→B` ≡ `¬B→¬A` (**유일하게 항상 동치**)
- 연쇄(가언삼단논법): `A→B`, `B→C` ⇒ `A→C`
- 핵심 기술: 모든 조건 옆에 **대우를 즉시 병기**한다. 거짓이 확정된 변수는 대우 사슬을 **역방향**으로 도미노시킨다.

**(3) 충분조건 vs 필요조건**
- `A→B`에서 A는 충분, B는 필요.
- "~여야 / ~해야 / ~없이는"이 붙은 쪽이 **필요조건 = 화살표 머리**.

### 3. 전략 (도식화 · 가지치기)
1. **발문 먼저** — "반드시 참 / 반드시 거짓 / 최소·최대" 중 무엇을 묻는지 확정.
2. **전 조건 기호화 + 대우 병기.**
3. **확정 단언부터 연쇄 발화** — 도미노로 결정되는 변수부터 채운다.
4. **갈림길은 가장 적게 갈리는 변수(2갈래)부터 분기** → 각 가지를 끝까지 밀어 모순이면 버린다(귀류).
5. **시각화** — 배치/순서/매칭은 **격자표**, 정언은 **벤다이어그램(전칭 먼저 빗금, 특칭 나중에 ×)**, 진실게임은 **진위표**.
6. **선지 대입은 최후 수단.**

### 4. 대표 함정 (오답이 노리는 논리오류)
- **역·이를 대우와 혼동** — `A→B`가 참이어도 역 `B→A`·이 `¬A→¬B`는 **도출 불가**. 참인 건 대우 `¬B→¬A`뿐. (후건긍정·전건부정의 오류)
- **충분/필요 뒤섞기** — "A여야 B"(`B→A`)를 `A→B`로 반대 기호화 → 결론이 통째로 뒤집힘.
- **양화사 혼동** — "어떤 S는 P 아님"(특칭 O)을 "모든 S는 P 아님"(전칭 E)으로 강화. **전칭에서 특칭(존재)을 함부로 끌어내지 말 것**(존재 함의 오류).
- **"또는"의 배타 해석** — 포함적 OR을 배타로 풀면 오답.
- **과잉 추론** — 조건에 없는 인과·순서를 가정하거나, "반드시 참"을 묻는데 "참일 수도 있는(미확정)" 선지를 고름.

## 실전 문제

### 문제 1. (조건문 연쇄 — 반드시 참인 것)

워크온의 만보기 앱 팀이 신규 기능 출시를 두고 다음 규칙에 합의했다.

- 푸시 알림을 켜면, 목표 달성률을 화면에 표시한다.
- 목표 달성률을 표시하면, 주간 리포트를 발송한다.
- 주간 리포트를 발송하면, 프리미엄 구독과 연동한다.
- 프리미엄 구독과 연동하려면, 다크 모드를 지원해야 한다.

이번 출시에서 팀은 **목표 달성률을 화면에 표시하지 않기로** 결정했다. 이 결정과 위 규칙으로부터 **반드시 참인 것**은?

① 푸시 알림을 켜지 않는다.
② 주간 리포트를 발송한다.
③ 다크 모드를 지원하면 푸시 알림을 켠다.
④ 프리미엄 구독과 연동하면 목표 달성률을 표시한다.
⑤ 주간 리포트를 발송하지 않으면 다크 모드를 지원하지 않는다.

## 해설

### 문제 1 — 정답 ①

**기호화** (P=푸시알림, G=목표달성률 표시, R=주간리포트, S=프리미엄 연동, D=다크모드)

| 조건 | 기호 | 대우(즉시 병기) |
|---|---|---|
| ① 푸시→달성률표시 | `P→G` | `¬G→¬P` |
| ② 달성률표시→리포트 | `G→R` | `¬R→¬G` |
| ③ 리포트→연동 | `R→S` | `¬S→¬R` |
| ④ 연동하려면 다크모드 필요 | `S→D` | `¬D→¬S` |

**단언:** 목표 달성률을 표시하지 않음 = `¬G`.

**연쇄(대우 사슬을 역방향으로 발화):**
`¬G` → (①의 대우 `¬G→¬P`) → **`¬P`** 확정.

여기서 멈춘다. `¬G`는 ②~④의 전건(G, R, S)이 아니므로 R·S·D는 **결정되지 않는다(자유)**. 즉 확정되는 것은 오직 `¬P`.

**선지 판정** (R·S·D가 자유임을 이용한 반례 탐색)
- **① `¬P`** — `¬G→¬P`로 도출. **반드시 참 ✓ (정답)**
- ② `R`(주간 리포트 발송) — R은 미확정. 발송 안 하는 모델도 가능 → 반드시 참 아님. ✗
- ③ `D→P` — **①(`P→G`)의 역 계열**. 다크모드는 지원(D)하면서 푸시는 끈(`¬P`) 모델이 조건을 모두 만족(반례) → ✗
- ④ `S→G` — 단언이 `¬G`이므로 만약 S가 참이면 `S→G`는 거짓. 프리미엄을 연동(S)하는 모델이 가능하므로 → ✗
- ⑤ `¬R→¬D` — **③(`R→S`)·④의 역 계열**. 리포트 미발송(`¬R`)이면서 다크모드 지원(D)인 모델이 조건을 만족(반례) → ✗

**회피 포인트:** `¬G`라는 **부정 단언**은 전건이 아니라 **후건의 부정**이므로, 정방향 연쇄가 아니라 **대우 사슬**로만 발화한다. ③·⑤처럼 "역(逆)"을 슬쩍 끼운 선지는 자유변수에 반례가 존재함을 떠올려 깬다.

### 문제 2 — 정답 ④

**기호화** (범인을 X로)

| 사람 | 진술 | 기호 |
|---|---|---|
| 갑 | "범인은 정" | X=정 |
| 을 | "범인은 갑" | X=갑 |
| 병 | "나는 아니다" | X≠병 |
| 정 | "범인은 갑이 아니다" | X≠갑 |

**1단계 — 모순쌍 포착:** 을("X=갑")과 정("X≠갑")은 **정면 모순**. 어떤 경우에도 **정확히 하나만 참**이다. 따라서 거짓 1명은 항상 이 둘(을·정) 안에서 1명 발생한다. 거짓은 총 1명뿐이므로 **갑·병은 둘 다 참**.

**2단계 — 갑·병이 참:**
- 갑 참 → **X=정** 확정.
- 병 참 → X≠병 (정과 모순 없음).

**3단계 — 진위표 채움(X=정 대입):**

| 가정 | 갑(X=정) | 을(X=갑) | 병(X≠병) | 정(X≠갑) | 거짓 수 | 판정 |
|---|---|---|---|---|---|---|
| **X=정** | 참 | 거짓 | 참 | 참 | **1** | ✓ 성립 |
| X=갑 | 거짓 | 참 | 참 | 거짓 | 2 | ✗ |
| X=을 | 거짓 | 거짓 | 참 | 참 | 2 | ✗ |
| X=병 | 거짓 | 거짓 | 거짓 | 참 | 3 | ✗ |

거짓이 정확히 1명인 경우는 **X=정** 한 가지뿐. 이때 거짓말한 사람은 **을**(혼자 거짓).

**∴ 범인 = 정 (정답 ④), 거짓 진술자 = 을.**

**오답이 범한 오류:**
- ② 을 — 을의 진술("X=갑")을 참으로 받아들이는 함정. 을은 유일한 거짓말쟁이다.
- ① 갑 — 을의 진술을 참으로 강화하면 빠진다(이때 갑·정 동시 거짓 → 거짓 2명, 메타조건 위반).
- ⑤ 알 수 없다 — 모순쌍 + 거짓 인원수 메타조건으로 **유일 확정**되므로 오답.

**회피 포인트:** 진실게임은 **모순쌍을 먼저 찾는다**. 모순쌍 밖의 진술(여기선 갑·병)이 자동으로 참이 되어 경우의 수가 급감한다. 전수 가정은 검산용으로만.

### 문제 3 — 정답 ③

**기호화** (A=만보 달성, B=프리미엄 구독, C=푸시 알림 ON)

| 조건 | 기호 |
|---|---|
| ① 모든 프리미엄은 푸시 ON | `B⊆C` (∀x: B→C) |
| ② 어떤 만보달성자는 프리미엄 | `∃x: A∧B` |
| ③ 푸시 ON은 모두 만보 달성 | `C⊆A` (∀x: C→A) |

전칭 ①③을 이으면 포함 사슬 **`B⊆C⊆A`**가 성립한다(프리미엄 ⊆ 푸시 ⊆ 만보달성).

**〈보기〉 판정**

**ㄱ. `∃x: A∧C` (어떤 만보달성자는 푸시 ON)**
- ②가 보장하는 인물(A∧B)을 앵커로 잡는다. 그는 B(프리미엄)이므로 ①(`B⊆C`)에 의해 **C(푸시 ON)**. 따라서 그는 A이면서 C.
- → **`∃x: A∧C` 반드시 참 ✓**

**ㄴ. `B⊆A` (모든 프리미엄은 만보 달성)**
- 포함 사슬 `B⊆C⊆A`에서 곧바로 `B⊆A`.
- → **반드시 참 ✓**

**ㄷ. `∃x: C∧¬B` (어떤 푸시 ON 사용자는 비구독)**
- 조건은 푸시 ON 사용자 중 **비구독자의 존재를 전혀 보장하지 않는다.** 모든 사용자가 (A,B,C 모두 참)인 모델, 즉 푸시 ON인 사람이 전부 프리미엄 구독자인 우주도 ①②③을 모두 만족한다(**반례**).
- → **반드시 참 아님 ✗** (존재를 새로 요구하는 함정)

**개인 타입 전수 검증** (제약 `B→C`, `C→A`를 만족하는 사람만 존재 가능)

| 타입 (A,B,C) | `B→C` | `C→A` | 허용? |
|---|---|---|---|
| (0,0,0) 아무것도 아님 | ✓ | ✓ | 허용 |
| (1,0,0) 만보만 | ✓ | ✓ | 허용 |
| (1,0,1) 만보+푸시(비구독) | ✓ | ✓ | 허용 |
| (1,1,1) 만보+구독+푸시 | ✓ | ✓ | 허용 |
| 그 외(B=1·C=0, C=1·A=0 등) | ✗ | | 불가 |

②(`∃A∧B`)를 만족하려면 (1,1,1) 타입이 **최소 1명** 필수. 이 앵커 1명만으로 ㄱ(A∧C)·ㄴ(B면 A)은 자동 충족. 반면 ㄷ(C∧¬B)에 해당하는 (1,0,1) 타입은 **0명일 수 있으므로** 반드시 참이 아니다.

**∴ 반드시 참 = ㄱ, ㄴ (정답 ③).**

**오답이 범한 오류:**
- ④·⑤(ㄷ 포함) — 전칭·존재 조건에서 **새로운 존재(비구독 푸시 사용자)를 강화**해 끌어낸 존재 함의 오류.
- ① ㄱ만 / ② ㄴ만 — 포함 사슬 `B⊆C⊆A`를 끝까지 잇지 않아 ㄱ 또는 ㄴ 하나를 놓침.

**회피 포인트:** **전칭(⊆)은 사슬로 이어 포함관계를 끝까지 확장**하고, **특칭(∃)은 앵커 1명**으로 잡는다. "어떤 ~가 있다"류 보기(ㄷ)는 **앵커가 그 속성을 보장하는지**, 아니면 **0명 반례 모델**이 가능한지를 본다.

## 한 줄 체크리스트
1. 발문 먼저(반드시참/거짓/최소·최대).
2. 전 조건 기호화 + **대우 즉시 병기**. 충분(꼬리)/필요(머리) 사수.
3. 부정 단언은 **대우 사슬**로 역방향 발화.
4. 진실게임은 **모순쌍 먼저** → 메타조건(거짓 n명) 대조.
5. 정언은 전칭 **사슬(⊆)**로 확장, 특칭은 **앵커**로 고정. 역·이는 참 아님(대우만 참).
